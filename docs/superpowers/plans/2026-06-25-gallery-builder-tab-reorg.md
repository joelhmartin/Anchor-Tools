# Anchor Gallery Builder — Tab Reorganization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize the Anchor Gallery builder tabs — delete Behavior, add a Video tab, move the three display toggles out of Content, tighten layout-scoped field filtering, and hide tabs that have no visible fields for the current layout.

**Architecture:** The builder is fully data-driven. Each setting in `get_setting_defs()` carries a `section` key that routes it to a tab; the tab list lives in `render_builder_after_title()`; `applies_to` arrays hide fields when the active `avg_layout` doesn't match. This plan changes `section` strings, the tab array, a few `applies_to` lists, and adds ~20 lines of JS to hide empty tabs. No meta keys change, so there is no data migration.

**Tech Stack:** Raw PHP (WordPress plugin), jQuery (IIFE, no build step for logic). Minified assets are regenerated by the release workflow; dev serves source under `SCRIPT_DEBUG`.

## Task 3: Hide tabs that have fields but none visible for the current layout

**Files:**
- Modify: `anchor-gallery/assets/admin.js` — `updateVisibility()` (~lines 135–161)

**Interfaces:**
- Consumes: builder-shell markup — tab buttons are `.anchor-builder__tab[data-tab="KEY"]`, panes are `.anchor-builder__pane[data-pane="KEY"]`, and each setting field is a `.anchor-builder__field` element inside its pane. These are emitted by `class-anchor-builder-shell.php` (`render()` and `render_field()`).
- Produces: after every visibility pass, a tab whose pane contains at least one `.anchor-builder__field` but zero currently-visible ones is hidden (both the tab button and the pane). Tabs whose panes contain no `.anchor-builder__field` at all (Content, Preset) are never hidden. If the active tab gets hidden, focus falls back to the first visible tab.

- [ ] **Step 1: Add a `hideEmptyTabs()` helper and call it from `updateVisibility()`**

In `anchor-gallery/assets/admin.js`, replace the end of `updateVisibility()`:

```php
      $row.toggle(visible);
    });
    applyForcedSettings();
  }
```

with:

```php
      $row.toggle(visible);
    });
    applyForcedSettings();
    hideEmptyTabs();
  }

  // Hide a builder tab when its pane has setting fields but none are currently
  // visible for the active layout. Panes with no .anchor-builder__field at all
  // (Content, Preset) are left alone. Keeps the active tab valid.
  function hideEmptyTabs(){
    var $builder = $('.anchor-builder').first();
    if (!$builder.length) return;
    var activeHidden = false;
    $builder.find('.anchor-builder__pane').each(function(){
      var $pane = $(this);
      var key   = $pane.attr('data-pane');
      var $tab  = $builder.find('.anchor-builder__tab[data-tab="' + key + '"]');
      var $fields = $pane.find('.anchor-builder__field');
      if (!$fields.length) { $tab.show(); return; }   // non-settings pane: never hide
      var anyVisible = $fields.filter(':visible').length > 0;
      $tab.toggle(anyVisible);
      if (!anyVisible && $tab.hasClass('is-active')) activeHidden = true;
    });
    // If the active tab was just hidden, activate the first visible tab.
    if (activeHidden) {
      var $first = $builder.find('.anchor-builder__tab:visible').first();
      if ($first.length) $first.trigger('click');
    }
  }
```

Note: `$fields.filter(':visible')` reflects the `$row.toggle(visible)` calls earlier in `updateVisibility()`, which run before `hideEmptyTabs()`. The `.anchor-builder__tab` click handler (in the shared `includes/builder/assets/builder.js`) handles activating the pane.
