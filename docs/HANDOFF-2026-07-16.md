## 2. Architecture facts you must know before touching the events module

- **occurrence = event post.** Never subdivide a post. Pick-one/recurring materialize **child event posts** via `Occurrences::reconcile()` (idempotent; roster-safe **soft-close** — a child with seats is never deleted). Multisession = one post + a repeatable session list (display/schema only).
- **Engine-owned meta** (`group_id`, `group_role`, `offering_dates`, `occurrence_key`, `occurrence_closed`, `recurrence`) is `show_in_rest => false` and **deliberately excluded** from `save_meta()`'s `$input` allow-list — written only by dedicated validated paths. Don't "helpfully" add them.
- **The event CPT editor is Gutenberg.** Two hard-won consequences: (a) classic `redirect_post_location` + `admin_notices` notices **do not surface** — use inline metabox validation; (b) `show_in_rest => true` on classic-metabox-only meta causes a **last-write-wins save race** that silently reverts user input.
- **Zero changes were made** to shipped seat/tier/product-sync/order-reconcile internals. Keep it that way.
- **Backward compat:** old external events store their link in legacy `_anchor_event_registration_url`; `Module::external_url()` falls back to it and the one-time migration maps it forward. Legacy `registration_type`/`registration_url` meta is **read-only kept** (UI removed) — never blank it.
- **Docs:** `anchor-events-manager/EVENTS.md`, `EMAILS.md`, `EVENTS-WOOCOMMERCE.md`; spec + plan in `docs/superpowers/`.

### B. Repo hygiene
- **`vendor/` is committed prod-only, but `composer install` (dev) dirties `vendor/composer/autoload_*` + `installed.*`.** These are **not** gitignored → a `git add -A` would ship dev-autoload into the plugin. Always stage scoped paths. Consider `.gitignore`ing the churn or restoring prod vendor after testing.
- **Not gitignored (should be?):** `.phpunit.result.cache`, `package-lock.json` (decide: commit it or ignore it), local `vendor/{bin,phpunit,…}` dev dirs.
- ✅ `*.min.js|css` **are** correctly gitignored and generated by the release Action — no stale committed artifacts (verified: 0 tracked).
- **Delete the dead `ACG_*` files** — `includes/class-acg-admin.php`, `class-acg-render.php` (+ siblings) are a never-loaded duplicate of the Schema module (grep-confirmed: never `require`d/instantiated). They were patched defensively in 3.9.13 so the latent XSS can't be revived, but they should just be deleted.

### C. Events module — deferred (all reviewed as non-blocking)
1. **Series-archive ordering**: the group parent's own `start_ts` isn't synced to its earliest live child, so an archive mixing multiple groups can sort wrong. Fix = sync parent `start_ts`/`end_ts` in `Occurrences::reconcile()`. (JSON-LD already computes it correctly.)
2. **N+1 queries** rendering large recurring groups (bounded — recurrence capped at 104 occurrences).
3. **Untrash/restore of a group parent** not handled (children stay soft-closed/trashed).
4. **Dead front-end notice path**: `queue_group_notice()` hooks `redirect_post_location`, which the front-end manager form never applies, and `admin_notices` never fires on the front end → that branch is dead code. Remove or wire it properly.
5. **offering↔recurring type switch + guard failure** leaves the old type's children live until the next valid save (conservative, self-heals — arguably correct).
6. **`wp_kses` strips `<!DOCTYPE>`** from *customized* email templates (unmodified ones never touch kses). Cosmetic — mail clients strip it anyway.
7. **Duplicate timezone logic** — extract a shared `Module::resolve_timezone()`; `Event_Schema::resolve_timezone()` duplicates it and could drift.
8. `occurrence_label()` has a dead `label` -meta branch (the engine only bakes the label into the post title).
9. **Empty-URL external event** renders an empty external shell; the pre-rework "Registration link unavailable." notice is now unreachable for external mode.
10. An **old internal event with a stray `registration_url`** derives as `external` (unusual data state; pre-existing derive logic).
11. `handle_registration()` guards on legacy `registration_type`, not `registration_mode` (pre-existing; only reachable via a crafted POST since the internal form isn't rendered for external mode).
12. JSON-LD **`availability` is event-level, not per-tier** (documented cost/complexity tradeoff).
13. Cosmetic: empty "Tickets / Pricing" postbox header shows under non-`wc` modes; the `[event_registration]` auto-append can double-render a child's form (both renders identically gated — no double-booking risk).
14. Legacy `registration_type`/`registration_url` meta is kept read-only for BC. A future full migration could drop them.

### D. Testing
- **`e2e/purchase.spec.js` fails environmentally** (guest cart empty at checkout → no billing form). **Proven pre-existing**: fails identically on pristine `origin/main`, the Phase-0 tip, and HEAD — it is *not* a regression from any of this work. Needs a fresh wp-env or the guest-cart/session flow hardened.
